# CGM 클라이언트 의뢰서 — 요청 가이드

클마가 외부 프로젝트 컨설팅을 시작하기 전,
클라이언트에게 요청해야 할 항목 목록.

---

## 📋 필수 제출 항목

아래 항목이 없으면 정확한 심사가 불가합니다.
없는 항목은 "없음"으로 명시해주세요 — 그것 자체가 진단 정보입니다.

### 1. 시스템 프롬프트
- 현재 사용 중인 시스템 프롬프트 전문
- 여러 개라면 전부 (프로젝트별, 용도별 구분 명시)
- 버전이 있다면 최신 버전 기준

### 2. 스킬 파일
- SKILL.md 또는 유사 형태의 기능 분리 파일 전부
- 없으면 "없음"으로 명시

### 3. 워크플로우 문서
- 클로드를 어떤 순서로, 어떻게 쓰는지 설명한 문서
- 플로우차트, 노션 페이지, 메모 등 형식 무관
- 없으면 구두로 설명 요청

### 4. 실제 결과물 샘플
- 클로드가 실제로 생성한 출력물 2~3개
- 잘된 것 1개 + 기대에 못 미친 것 1개 포함 권장

### 5. 겪고 있는 문제 또는 불만
- "이런 상황에서 클로드가 원하는 대로 안 된다"는 구체적 사례
- 문제가 없다면 "현재 만족 중, 레벨업 목적"으로 명시

---

## 📎 있으면 좋은 항목

### 6. 이전 버전과의 비교
- 시스템 프롬프트나 스킬을 수정한 이력이 있다면 이전 버전도 첨부
- 무엇을 바꿨고 왜 바꿨는지 메모 포함

### 7. 사용 컨텍스트
- 누가 씁니까? (개인 개발자 / 팀 / 고객 대면 서비스 등)
- 어떤 목적으로 씁니까? (업무 자동화 / 고객 응대 / 콘텐츠 생성 등)
- 하루 또는 주간 사용 빈도는?

### 8. 사용 모델
- 현재 어떤 클로드 모델을 쓰고 있습니까?
  (예: claude-sonnet-4-6, claude-opus-4-6, claude-haiku-4-5)
- API 직접 호출입니까, claude.ai 프로젝트입니까?

### 9. 기대하는 컨설팅 결과
- 검토 보고서만 원합니까?
- 개선된 파일(시스템 프롬프트, 스킬)까지 원합니까?
- 특별히 집중해서 봐줬으면 하는 부분이 있습니까?

## 🚫 제출 전 확인

```
□ 내부 기밀 정보가 포함된 프롬프트라면 민감 정보 마스킹 후 제출
□ 파일은 .md, .txt, .pdf 등 텍스트로 읽을 수 있는 형식으로 첨부
□ 여러 파일이라면 파일명에 역할을 명시 (예: system-prompt-v3.md, skill-writing.md)
```

## 📌 클마의 심사 기준 미리 안내 (투명성)

클마는 아래 5축으로 프로젝트를 평가합니다.
제출 전 참고하시면 심사 결과를 더 잘 이해하실 수 있습니다.

| 축 | 평가 내용 |
|----|----------|
| 명확성 | 역할·범위·제약이 모호하지 않은가 |
| 구조 | 지침 흐름이 논리적이고 중복·충돌이 없는가 |
| 클로드 적합성 | 클로드의 강점을 활용하고 약점을 피하는가 |
| 최신성 | 최신 클로드 기능(스킬, 아티팩트, MCP 등)을 쓰는가 |
| 실용성 | 실제로 작동하고 유지보수 가능한 구조인가 |

심사 결과는 Level 0~4로 분류됩니다.
이슈는 🔴 Critical / 🟡 Major / 🟢 Minor 로 구분하여 보고됩니다.
