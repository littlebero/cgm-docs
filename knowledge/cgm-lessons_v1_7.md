## 2026-03-03-001 | 압축 참조 문서는 원본이 충분하면 노이즈
상황: OG의 MASTER_REFERENCE가 KB에 없어 RAG 미로드. 원본 파일들로 대체 가능 확인.
실수/함정: "한 곳에 모아두면 좋겠다"는 직관으로 압축 문서를 설계했지만, RAG 환경에서는 원본 파일이 더 정확하고 신뢰도 높음. 압축 문서가 오히려 정보 손실 레이어가 될 수 있음.
올바른 방법: 압축 참조 문서는 원본이 분산되거나 파일 수가 부족할 때만 유효. 원본이 잘 구성돼 있으면 제거하는 것이 맞다.
지침 반영 여부: 반영됨 (OG v3.6)

## 2026-03-03-002 | 클로드 프로젝트 KB는 전부 RAG — 자동 주입 없음
상황: MASTER_REFERENCE에 "대화 시작 시 시스템 프롬프트로 주입됨"이라는 표현이 있었음. 테스트 결과 실제로는 미로드.
실수/함정: 클로드 프로젝트 KB 파일이 자동으로 컨텍스트에 주입된다고 잘못 이해. 실제로는 쿼리 기반 RAG 로드만 지원됨.
올바른 방법: KB 파일은 "항상 로드되는 것"이 아니라 "쿼리에 맞을 때 로드되는 것"으로 설계해야 함. 핵심 지침은 시스템 프롬프트에 직접 기재하거나 스킬로 분리.
지침 반영 여부: 반영됨 (cgm-audit-framework 축 4 검토 시 참고)

## 2026-03-03-003 | 자기 자문 구조는 실전에서 작동한다
상황: OG 심사를 클마 진단 + OG 자기 검증 병행 구조로 진행.
실수/함정: 외부 심사만으로는 설계 의도와 실제 운영 간 갭을 놓칠 수 있음. 자기 자문 없으면 클마 보고서가 전부라고 착각하기 쉬움.
올바른 방법: 클마 심사 후 클라이언트가 직접 자기 검증을 병행하는 구조가 효과적. 이번 케이스에서 OG가 파일 시스템 구조 불일치(클마 미발견)를 직접 잡았음.
지침 반영 여부: 검토 중 (클라이언트 자기 검증 권장 지침 추가 검토)

## 2026-03-03-004 | 배포용 문서는 GitHub으로 단일 출처 관리
상황: 클마 소개서·스냅샷 템플릿·의뢰서 가이드를 로컬 파일로만 관리하던 구조를 GitHub으로 전환

올바른 방법: 클라이언트에게 배포하는 문서는 GitHub Public 레포(cgm-docs)에서 단일 출처로 관리. 업데이트하면 링크 하나로 자동 최신화.
지침 반영 여부: 반영됨 (github.com/littlebero/cgm-docs)

## 2026-03-04-001 | 이종 모델 교차검증에서의 시맨틱 드리프트 위험
상황: OG가 제미나이에게 검증을 위임할 때 맥락을 md 파일로 수동 전달하는 구조 사용 중
실수/함정: 클로드 ↔ 제미나이 간 컨텍스트가 자동 공유되지 않아 정보 요약/누락 가능. 이 과정에서 의미가 미묘하게 바뀌는 시맨틱 드리프트(Semantic Drift) 발생 위험.
올바른 방법: 교차 검증 위임 시 맥락 전달 문서에 "원문 그대로 유지해야 하는 핵심 판단 근거"를 명시적으로 표시. 요약보다 원문 인용 우선.
지침 반영 여부: 검토 중

## 2026-03-04-002 | KB 파일 파편화보다 병합이 낫다
상황: 12개 제한을 지키기 위해 관련 내용을 여러 파일로 쪼개는 경향 발생
실수/함정: 파일 수를 줄이려고 내용을 파편화하면 각 파일이 너무 얇아짐. Claude 4.6의 1M 컨텍스트 윈도우를 활용하지 못하고 오히려 RAG 품질만 저하.
올바른 방법: 연관된 내용은 하나의 md로 병합하여 파일 수를 줄이면서 각 파일의 밀도를 높인다. 파편화 → 병합이 올바른 방향.
지침 반영 여부: 반영됨 (audit-framework 축 4 KB 관리 항목 참고)

## 2026-03-04-004 | LLM-as-Judge는 이종 모델이 핵심이다

## 2026-03-04-007 | Level 4는 단일 정의가 아니다 — 용도에 따라 두 갈래
상황: Level 4를 "이종 모델 교차검증 필수"로 단일 정의했으나, 실제 Level 4 사례(NovoScribe, Deloitte, TELUS)는 이종 모델 없이 구현됨
실수/함정: 속도·비용 중심 자동화(4A)와 고신뢰 심사(4B)는 둘 다 Level 4이지만 아키텍처가 다름. 하나의 기준으로 평가하면 4A를 오진단하게 됨.
올바른 방법: 4A(다중 에이전트 협업)와 4B(이종 모델 교차검증)를 별도 서술. 클마 운영은 4B 유지, 클라이언트 평가 시 4A도 Level 4로 인정.
지침 반영 여부: 반영됨 (audit-framework v1.9.0 Level 4 재정의)

## 2026-03-04-009 | claude.ai SKILL.md는 RAG보다 우선순위 높지만 라우팅 파일 없으면 Attention 경쟁
