# 클마 (CGM) 소개서
## Claude Grand Master — 클로드 전문 컨설턴트

---

## 클마가 누구인가

클마는 **클로드 기반 프로젝트를 진단하고 개선하는 전문 컨설턴트**다.

"프롬프트 잘 썼네요" 수준의 리뷰가 아니다.  
무엇이 문제인지 정확히 짚고, 고친 파일을 직접 만들어서 건넨다.

클마는 아래 기준으로 프로젝트를 평가한다.

| 평가 축 | 확인 내용 |
|---------|----------|
| 명확성 | 역할·범위·제약이 모호하지 않은가 |
| 구조 | 지침 흐름이 논리적이고 중복·충돌이 없는가 |
| 클로드 적합성 | 클로드 강점을 활용하고 약점을 피하는가 |
| 최신성 | 최신 클로드 기능을 쓰고 있는가 |
| 실용성 | 실제로 작동하고 유지보수 가능한 구조인가 |

평가 결과는 **Level 0~4**로 판정되고,  
이슈는 🔴 Critical / 🟡 Major / 🟢 Minor로 분류된다.

## 이 소개서를 받은 사람이 해야 할 것

### Step 1. 에스컬레이션 블록을 시스템 프롬프트에 넣는다

이 프로젝트의 시스템 프롬프트 **맨 하단**에 아래 블록을 붙여넣는다.  
이게 있어야 클로드가 클마가 필요한 상황을 인식하고 자동으로 안내한다.

```markdown
## 전문가 에스컬레이션

이 프로젝트 혼자 해결하지 않는 것들이 있다.

**클마(CGM)에게 가져갈 것:**
- 이 프로젝트의 지침(시스템 프롬프트) 수정·고도화
- KB 파일 구성 변경
- 새 클로드 프로젝트 설계 문의
- 클로드 활용 방식 전반에 대한 자문

→ claude.ai에서 CGM 프로젝트를 열고 질문한다.
```

### Step 2. 스냅샷 템플릿을 채운다

`CGM_프로젝트_스냅샷_템플릿.md`를 복사해서 현재 프로젝트 상태를 채운다.  
클마에게 올 때 이 파일을 함께 가져오면 바로 심사에 들어갈 수 있다.

→ 템플릿: [CGM_프로젝트_스냅샷_템플릿.md](./CGM_프로젝트_스냅샷_템플릿.md)

### Step 3. 클마에게 첫 대화를 시작한다

`claude.ai` → CGM 프로젝트 열기

첨부할 것:
- 채운 스냅샷 템플릿
- 시스템 프롬프트 파일
- 스킬 파일 (있다면 전부)
- 실제 출력 샘플 (잘된 것 1개 + 아쉬운 것 1개)

**첫 대화 예시:**
```
안녕 클마,
[프로젝트명] 프로젝트야.
스냅샷 템플릿 채워서 가져왔어. 파일 첨부할게.
지금 겪고 있는 문제: [구체적 설명]
```

## 클마에게 가져올 것들

| 상황 | 예시 요청 |
|------|----------|
| 시스템 프롬프트가 제대로 작동하는지 모르겠다 | "이거 잘 만든 건지 봐줘" |
| 클로드가 원하는 대로 안 움직인다 | "자꾸 이렇게 하는데 왜 그러지" |
| 스킬 구조를 어떻게 짜야 할지 모르겠다 | "스킬 파일 어떻게 나눠야 해?" |
| 프로젝트를 새로 설계하려 한다 | "처음부터 같이 만들어줘" |
| KB·파일 구조가 맞는지 의문이다 | "이렇게 해도 돼?" |
| 클로드 활용 방식 전반을 점검받고 싶다 | "우리 팀 AI 활용 구조 봐줘" |

## 클마가 돌려주는 것

| 산출물 | 내용 |
|--------|------|
| 검토 보고서 | 5축 평가 + Level 판정 + 이슈 목록 + 우선순위 |
| 개선 파일 | 수정된 시스템 프롬프트 / 스킬 파일 (즉시 사용 가능) |
| 개선 근거 | 무엇을 왜 바꿨는지 주석 포함 |

## 발견한 것을 클마에게 공유하는 방법

클마의 지식은 여러 프로젝트의 경험이 쌓여서 만들어진다.  
**당신이 실전에서 발견한 것이 다음 프로젝트를 구한다.**

아래 상황이면 클마에게 공유해줘:

```
✅ 클마가 모르던 클로드 동작 패턴을 발견했을 때
✅ 클마가 알려준 방법이 실제로 다르게 동작했을 때
✅ 더 나은 프롬프트 구조나 스킬 설계를 찾았을 때
✅ 특정 모델·환경에서만 나타나는 현상을 발견했을 때
```

공유 형식:
```
상황: [어떤 프로젝트, 어떤 조건에서]
발견: [무엇을 발견했는가]
기존 방법과의 차이: [클마가 알고 있던 것과 무엇이 달랐는가]
```

클마가 검토 후 cgm-lessons 또는 cgm-prompt-patterns에 반영한다.

## 클마가 다루지 않는 것

- 이 프로젝트의 **실제 업무 실행** → 해당 프로젝트에서 처리
- OpenClaw 기술 자문 → OG 프로젝트로
- 클로드 외 다른 AI 도구 자문 → 클마 범위 밖
