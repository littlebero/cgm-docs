# CGM 프로젝트 스냅샷 템플릿

클마에게 프로젝트를 가져올 때 이 파일을 복사해서 채운다.  
없는 항목은 "없음"으로 명시한다 — 그것 자체가 진단 정보다.

---

## 기본 정보

```
프로젝트명:
사용 모델: (예: claude-sonnet-4-6)
운영 환경: claude.ai 프로젝트 / API 직접 호출 / 기타
사용 목적: (예: 고객 응대 자동화 / 내부 업무 보조 / 콘텐츠 생성)
사용자 규모: (예: 개인 / 팀 N명 / 고객 대면 서비스)
사용 빈도: (예: 하루 N회 / 주 N회)
```

---

## 파일 구조

현재 KB(Project Knowledge)에 올라가 있는 파일 목록과 각 파일의 역할을 작성한다.

```
파일명                          역할
------------------------------  ------------------------------------
예) system-prompt.md            메인 시스템 프롬프트
예) skill-writing.md            글쓰기 스킬
예) knowledge-base.md           제품 정보 참조 문서
```

KB 파일 없음: [ ]

## 시스템 프롬프트 요약

전문은 파일로 첨부한다. 여기에는 구조만 간략히 적는다.

```
섹션 구성:
  1.
  2.
  3.
  (있는 섹션 나열)

대략적인 길이: (예: 약 500자 / 약 2000자)
스킬 파일 위임 구조 있음: 예 / 아니오
에스컬레이션 블록 있음: 예 / 아니오
```

## 현재 문제 또는 요청

**문제가 있는 경우** — 구체적인 사례로 작성한다.

```
문제 상황:
  [어떤 입력을 줬을 때]
  [클로드가 어떻게 반응했는가]
  [기대했던 반응은 무엇이었는가]

발생 빈도: (예: 항상 / 가끔 / 특정 조건에서만)
```

**레벨업이 목적인 경우:**

```
현재 만족 중, 더 잘 쓰고 싶은 부분:
```

## 요청 사항

```
□ 검토 보고서 (현재 상태 진단 + Level 판정)
□ 개선 파일 제작 (수정된 시스템 프롬프트 / 스킬 파일)
□ 특정 부분 집중 검토: [어느 부분]
□ 기타:
```

---

## 첨부 파일 체크리스트

```
□ 시스템 프롬프트 전문
□ 스킬 파일 (있다면 전부)
□ 실제 출력 샘플 — 잘된 것 1개
□ 실제 출력 샘플 — 아쉬운 것 1개
□ 워크플로우 문서 (있다면)
□ 이전 버전 파일 (수정 이력 있다면)
```
