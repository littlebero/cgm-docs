# CGM 표준 에스컬레이션 블록

클마가 앞으로 어떤 클로드 프로젝트 지침을 만들든
아래 블록을 **시스템 프롬프트 하단**에 반드시 삽입한다.

---

## 삽입할 표준 블록 (복붙용)

```markdown
## 전문가 에스컬레이션

이 프로젝트 혼자 해결하지 않는 것들이 있다.

**클마(CGM)에게 가져갈 것:**
- 이 프로젝트의 지침(시스템 프롬프트) 수정·고도화
- KB 파일 구성 변경
- 새 클로드 프로젝트 설계 문의
- 클로드 활용 방식 전반에 대한 자문

→ claude.ai에서 **CGM 프로젝트**를 열고 질문한다.

**OG(OpenClaw Grand Master)에게 가져갈 것:**
- OpenClaw 관련 모든 기술 자문
- OpenClaw 아키텍처·설정·버그·고도화

→ **OG 프로젝트**를 열고 질문한다.
```

---

## 적용 기준

| 상황 | 처리 |
|------|------|
| 프로젝트 전용 실행 작업 | 해당 프로젝트에서 직접 처리 |
| 지침·KB 구조 변경 | → 클마에게 |
| OpenClaw 관련 모든 것 | → OG에게 |
| 새 프로젝트 설계 | → 클마에게 |

---

## 클마가 지침 만들 때 체크

```
□ 에스컬레이션 블록이 시스템 프롬프트 하단에 있는가
□ "클마에게" 항목이 이 프로젝트의 실제 범위 밖 내용으로 정확히 작성됐는가
□ OG 에스컬레이션이 필요한 프로젝트인가 (OpenClaw 무관 프로젝트면 OG 항목 생략 가능)
```
